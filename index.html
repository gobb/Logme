    What Is It?

    Logme is a full-featured logging framework for PHP 5.4+.

    Y U NO LIKE STATIC???

    It makes sense to have only a single logging object in your application. The problem with traditional logging libraries is that they usually achieve this state by storing the logging object in a static variable or wrapping it in an evil Singleton object. These methods are fancy ways of saying, "I'm storing" my logger in a global variable. And we all know that globals are evil.

    Logme improves on this model by expecting developers to employ Dependency Injection to provision application classes for logging. The result is more testable, maintable code.

    Features

    Log Handler Chain

    Logme uses a variation on the Chain of Responsiblity pattern to attach multiple log handlers, giving the developer full control over what events are logged and to which destinations. Consider the needs of a typical web application:

    You wish to write low-priority notices or warnings to a simple log file but direct more severe log events like uncaught exceptions or critical system failures to a database or REST API for appropriate handling and notification.

    Logme turns this kind of complex logging into a single orthogonal method call.

    Extensible Built-in Handlers

    Logme comes packaged with several built-in handlers for logging to files, database resources and the console. More importantly, Logme makes creating your own custom handlers a triviality.

    Custom Filters

    Handlers can implement fully-customizable filters for fine-grained control over which events are actually logged.

    Log Output Formatting

    Formatter objects expose a standardized interface for formatting log values.
